## 🎯 优先级决策依据

### 为什么Agent自动创建是P0？
- 用户每次创建员工都要手动编辑配置文件
- 容易出错，体验极差
- 是系统自动化程度的核心指标

### 为什么Agent执行闭环是P0？
- 没有闭环，任务分配只是数据库状态更新
- Agent不会真正工作，系统失去意义
- 这是从"玩具"到"工具"的关键一跃

### 为什么任务流是P1而非P0？
- 单任务单员工模型已能满足基础需求
- 任务流需要更复杂的状态机设计
- 建议在Agent执行闭环稳定后再实现

### 为什么PostgreSQL是P0？
- SQLite不支持并发写入
- 外网访问场景需要更高性能
- 数据安全和备份需求

## 💡 技术实现要点

### Agent自动创建
```python
# 伪代码
class AgentLifecycleService:
    def create_agent(self, name, position):
        # 1. 备份配置
        backup_openclaw_json()
        
        # 2. 修改配置
        config = load_openclaw_json()
        config['agents']['list'].append({
            'id': agent_id,
            'name': name,
            'workspace': f'~/.openclaw/workspace/{agent_id}'
        })
        save_openclaw_json(config)
        
        # 3. 创建workspace
        create_workspace(agent_id)
        generate_soul_md(name, position)
        
        # 4. 返回提示
        return {"success": True, "need_restart": True}
```

### Agent执行闭环
```python
# 伪代码
class TaskExecutionService:
    async def execute_task(self, task_id, agent_id):
        # 1. 获取Agent的OpenClaw Agent ID
        agent = get_agent(agent_id)
        
        # 2. 构建消息
        message = build_task_message(task)
        
        # 3. 发送给Agent
        result = await sessions_send(
            session_key=agent.agent_id,
            message=message
        )
        
        # 4. 等待回调（异步）
        # Agent完成后调用 /api/tasks/{id}/complete
        
        # 5. 更新任务状态
        update_task_status(task_id, "executing")
```

### 任务流状态机
```
[created] → [step1_pending] → [step1_executing] → [step1_review]
                                                    ↓
                                        ┌─────[step1_rework]────┐
                                        ↓                       │
                                    [approved]              [rejected]
                                        ↓
                                   [step2_pending] → ... → [final_review]
                                                                ↓
                                                           [completed]
```
