## 💡 用户反馈：协作机制设计

### 当前模型的局限

> "任务单独分配给一个员工的做法违背了多 Agent 合作的初衷"

### 新设计：任务流 + 审批流

#### 核心概念

```
任务 (Task) 
    ↓
任务流 (Workflow) - 由 Partner 根据任务内容和核心员工编写
    ↓
多个步骤 (Steps) - 按顺序执行
    ↓
轮流工作 + 评分机制
```

#### 执行流程

```
┌─────────────────────────────────────────────────────────────┐
│  步骤1: 员工A执行 → 输出结果                                 │
│           ↓                                                 │
│  步骤2: 员工B获取A的输出 → 评分                              │
│           ↓                                                 │
│     ┌──────────┐                                            │
│  合格? → B在A的基础上继续 → 传递给C                         │
│     └──────────┘                                            │
│  不合格? → 返工给A，附上修改意见                             │
│           ↓                                                 │
│  最后一步: 高权限员工/Parter 最终审批 → 报告给用户            │
│           ↓                                                 │
│  用户评价 → 评分/返工/反馈                                   │
└─────────────────────────────────────────────────────────────┘
```

#### 激励机制

| 评分类型 | 影响对象 | 目的 |
|----------|----------|------|
| 最终评分 | 所有参与员工 | 团队协作质量 |
| 下游评分 | 上游员工 | 互相监督质量 |
| 返工次数 | 该步骤员工 | 减少返工，提高效率 |

**核心**: 评分影响奖励，激励高质量协作。

#### 项目机制

```
项目 (Project)
├── 项目负责人（高权限员工）
├── 根据项目内容拆解任务
├── 定时发布任务推进项目
├── 评估任务完成后的项目进展
└── 项目输出由用户评价
```

#### 触发机制（重要）

**OpenClaw Agent 是触发制**，需要在关键场景主动触发：

| 场景 | 触发方式 |
|------|----------|
| 任务到达员工 | `sessions_send` 通知 |
| 上一步完成 | 自动触发下一步员工 |
| 返工请求 | 触发被返工员工 |
| 定时任务 | 项目负责人主动发起 |
| 项目里程碑 | 触发报告生成 |
| 用户反馈 | 触发相关人员 |

**例外**: 项目管理员工可以设置定时任务主动推进项目。

## 📊 系统架构演进

### 当前 (v0.3.0)

```
用户 → Partner → 任务 → 单个员工
                 ↓
              执行 → 完成 → 报告
```

### 新设计 (v0.4.0+)

```
用户 → Partner → 项目 → 拆解任务
                          ↓
                    任务流编排
                          ↓
            步骤1 → 步骤2 → 步骤3 → 最终审批
              ↓       ↓       ↓         ↓
            评分    评分    评分      用户评分
              ↓       ↓       ↓         ↓
            动态奖励分配（基于评分链）
```

## 📝 关键设计原则

1. **触发制优先**: Agent 不主动轮询，由系统事件触发
2. **质量驱动**: 评分机制确保输出质量
3. **透明激励**: 奖励与评分直接挂钩
4. **灵活编排**: Partner 负责任务流编排，适应不同场景
5. **项目视角**: 任务不是孤立存在，而是项目的一部分
