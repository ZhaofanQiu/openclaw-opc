# OpenClaw OPC - UI功能缺失项整合设计方案

## 设计原则

1. **场景化整合** - 功能出现在用户需要它的场景，而非独立页面
2. **隐喻一致性** - 保持"CEO管理面板"和"星际工作室"的概念
3. **渐进式披露** - 次要功能折叠或放入详情页，不干扰主流程
4. **像素办公室联动** - Dashboard与像素办公室形成互补，而非重复

---

## 1. 交易记录查看 (P0) - 「公司财务」概念

### 产品逻辑
交易记录不是独立的"表格页面"，而是公司财务的一部分。用户想看到：
- "我的钱去哪了？"
- "哪个员工花了多少？"
- "哪类任务最烧钱？"

### UI实现方案

**A. Dashboard预算卡片增强（主入口）**
```
📊 总预算: 10,000 OC币
💸 已使用: 3,240 OC币 (32%)
📈 查看明细 →  （新增链接）
```
点击后展开/弹出「财务报告」面板，而非跳转新页

**B. 财务报告面板设计（内嵌Modal/抽屉）**
- **顶部筛选**: 时间范围(7天/30天/全部) | 员工筛选 | 任务类型
- **可视化**: 饼图（支出构成）+ 折线图（消费趋势）
- **交易列表**: 简化表格，关键字段：
  - 时间 | 员工头像+名字 | 任务名称(截断) | 金额 | 操作类型图标
- **导出按钮**: CSV导出（放在角落，不显眼）

**C. 员工详情页关联**
点击某员工 → 展开详情 → 显示"该员工消费记录"子列表

### 与现有UI的整合点
- 复用现有统计卡片样式
- 使用已有的Chart.js图表组件
- 保持深色主题一致性

---

## 2. API Key管理 (P1) - 「系统管理员」概念

### 产品逻辑
API Key管理不是普通用户的日常操作，是"系统管理员/CEO的技术后台"。应该：
- 不放在主Dashboard（避免普通用户困惑）
- 放在设置/配置区域
- 视觉上区分：这是"系统设置"而非"业务操作"

### UI实现方案

**A. 系统配置区域扩展（现有config卡片内）**
```
⚙️ 系统配置 (现有折叠面板)
├── 任务超时时间
├── Token换算比例
├── 预算警告阈值
├── ────────────────
├── 🔐 API访问管理  → （新增折叠项）
│   ├── 当前有效Key: 2个
│   ├── [查看管理按钮] → 展开Key列表
```

**B. Key管理子面板（内嵌展开）**
- 简洁列表：名称 | 创建时间 | 权限标签(只读/读写) | 操作按钮(撤销/轮转)
- **创建按钮**: 弹出创建表单（名称+权限选择）
- **复制提示**: 创建后一次性显示Key，带复制按钮

**C. 安全提醒**
- 面板顶部黄色警告条："API Key 具有访问系统权限，请妥善保管"

### 为什么这样设计
- 不新增导航/页面，保持左侧导航简洁
- 放在"系统配置"符合心智模型
- 默认折叠，不干扰普通用户

---

## 3. 消息通信中心 (P1) - 「员工对话」概念

### 产品逻辑
消息不是"聊天软件"，是"CEO给员工下达指令"和"员工汇报"的渠道。应该：
- 与员工绑定，而非独立消息列表
- 突出"任务相关消息"，弱化闲聊
- 在任务场景中出现，而非独立入口

### UI实现方案

**A. 员工详情页「对话」Tab（新增）**
点击员工 → 展开详情 → 两个Tab：「档案」/「对话记录」

对话记录设计：
- 简短消息气泡列表（最新10条）
- 底部快速发送框：「发送消息」输入框 + 发送按钮
- 标记为"已读"按钮（如果Partner需要确认）

**B. 任务卡片「沟通」按钮（任务列表）**
```
📋 任务: 修复登录Bug
    👤 分配给: 小K
    💰 预算: 500 OC币
    [开始任务] [查看进度] [💬 沟通] ← 新增
```
点击打开小弹窗，显示与该任务相关的消息记录

**C. 全局「通知中心」增强（右上角铃铛）**
现有通知只是被动接收，增强为：
- 通知列表底部加「查看全部消息」
- 区分：系统通知 vs 员工消息
- 消息可点击，跳转对应员工的对话页

### 为什么这样设计
- 消息依附于"员工"和"任务"，符合产品隐喻
- 避免做一个独立的"聊天页面"破坏游戏感
- 与现有通知铃铛整合，减少入口

## 4. 熔断解决操作 (P1) - 「危机处理」概念

### 产品逻辑
熔断是"公司危机"，解决操作应该是：
- 醒目但不过度恐慌
- 提供清晰的决策路径（追加预算/换人/拆分/暂停）
- 与熔断警报联动

### UI实现方案

**A. 熔断警报卡片（现有）增强**
当前熔断警报显示为静态文本，改造为：
```
🔥 预算熔断警报
─────────────────
员工: 小K
任务: 修复登录Bug
已用/预算: 550/500 OC币 (110%)

[💰 追加预算] [🔄 换人重做] 
[✂️ 拆分任务] [⏸️ 暂停任务]
```

**B. 解决方案Modal（点击后弹出）**
每种解决方案对应一个简洁的确认弹窗：
- **追加预算**: 输入框（建议金额 = 已超支 + 20%缓冲）+ 原因说明
- **换人重做**: 下拉选择其他员工 + 原因说明
- **拆分任务**: 子任务列表（可编辑）+ 预算重新分配
- **暂停任务**: 确认提示（确认后任务状态变为暂停）

**C. 操作反馈**
- 所有操作后显示Toast通知："✓ 已追加预算 200 OC币"
- 刷新相关卡片数据

### 为什么这样设计
- 熔断处理直接内嵌在警报卡片，无需跳转
- 四个按钮直观呈现所有选项
- 保持紧迫感但提供清晰的解决路径

## 5. 技能管理 (P2) - 「人才库」概念

### 产品逻辑
技能不是"标签管理"，是"公司的人才能力图谱"。应该：
- 在员工招聘/分配场景中出现
- 可视化展示（词云或技能雷达图）
- 与任务匹配度联动

### UI实现方案

**A. 员工档案「技能」区域优化（现有）**
现有技能是文字列表，改为视觉化：
- 技能标签带颜色区分（技术/创意/管理/其他）
- 悬停显示"擅长度"（如果有数据）

**B. 任务分配「匹配度」提示**
分配任务时，在员工选择下拉框中显示：
```
小K - 前端开发     [匹配度 95% ⭐]
小明 - 全栈开发   [匹配度 70%]
```

**C. 全局技能统计（可选）**
在Dashboard底部新增小卡片：
```
🛠️ 公司技能图谱
前端 ████████░░ 80%
后端 ██████░░░░ 60%
设计 ████░░░░░░ 40%
```
点击展开完整技能列表（仅当有空间时）

### 为什么这样设计
- 技能是"员工的属性"，不应独立管理
- 在"任务分配"场景展示匹配度最有价值
- 避免做一个"技能CRUD页面"，那是数据库管理而非产品功能

## 6. Partner操作界面 (P2) - 「CEO助理」概念

### 产品逻辑
Partner Agent是"CEO的助理"，不是普通员工。应该：
- 突出其"特殊身份"
- 操作是"高层决策"（雇佣、全局分配）
- 普通员工操作与之区分

### UI实现方案

**A. Partner专属区域（Dashboard顶部）**
```
┌─────────────────────────────────────┐
│ 👑 Partner 状态: 🟢 健康            │
│ 今日任务: 分配 3 个 | 完成 2 个     │
│                                     │
│ [🤝 雇佣新员工] [📋 全局任务分配]  │
└─────────────────────────────────────┘
```

**B. 「雇佣新员工」流程**
点击后弹出：
- 选择OpenClaw Agent（下拉列表）
- 设置职位名称
- 设置初始预算
- 分配初始技能（标签输入）

**C. 「全局任务分配」流程**
- 显示待分配任务列表
- Partner建议的最佳员工（带匹配度）
- 一键全部分配 或 逐个确认

**D. Partner健康度详情（点击状态展开）**
- 心跳时间线
- 最近分配的任务
- 性能指标（成功率、平均耗时）

### 为什么这样设计
- 将Partner操作放在顶部，体现其"高层"地位
- 雇佣和全局分配是CEO决策，不是日常操作
- 与员工区域视觉区分（不同背景色/边框）

## 实施优先级建议

### 第一阶段（v0.3.x 快速补全）
1. **熔断解决操作** - 修复现有按钮，让核心保护机制可用
2. **交易记录** - 内嵌财务面板，满足预算追溯需求

### 第二阶段（v0.4.x 体验优化）
3. **消息通信** - 员工详情页对话Tab + 任务沟通按钮
4. **API Key管理** - 系统配置内嵌面板

### 第三阶段（v0.5.x 功能增强）
5. **Partner界面** - 如果Partner是核心卖点，提前到第二阶段
6. **技能管理** - 匹配度提示优先，技能图谱可选

## 技术实现要点

### 新增API调用封装
所有功能复用现有的 `Auth.authFetch()` 方法，保持认证一致性。

### 组件复用
- Modal: 复用现有的 `modal-overlay` 和 `modal` 样式
- 列表: 复用现有的 `agent-item` 样式
- 图表: 复用Chart.js配置
- 表单: 复用修复后的深色主题表单样式

### 状态管理
新增全局状态：
```javascript
// 在App对象中新增
App.state = {
    transactions: [], // 交易记录缓存
    messages: {},     // 员工消息缓存 {agentId: [messages]}
    apiKeys: [],      // API Key列表
    partnerStatus: {} // Partner状态
};
```

### 路由/导航
**不新增导航项**，所有功能通过：
- 现有卡片的「展开/查看详情」入口
- Modal弹窗
- 抽屉式侧边栏（可选，用于复杂页面如财务报告）

- [ ] API Key管理不会干扰普通用户的Dashboard使用
- [ ] 消息功能自然地出现在员工和任务场景中
- [ ] 所有新增功能保持深色主题一致性
