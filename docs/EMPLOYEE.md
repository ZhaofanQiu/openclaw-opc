# 员工系统设计 (Employee System)

## 员工模型

### 基础属性

```typescript
interface Employee {
  // 基础信息
  id: string;                    // 唯一标识
  name: string;                  // 员工姓名
  emoji: string;                 // 头像 emoji
  avatar_url?: string;           // 自定义头像（可选）
  
  // 职位信息
  position_level: 1 | 2 | 3 | 4 | 5;  // 职级
  position_title: string;        // 职位名称
  
  // OpenClaw 关联
  agent_id: string;              // 关联的 OpenClaw Agent ID
  session_key?: string;          // 当前会话密钥
  
  // 预算系统
  monthly_budget: number;        // 月薪（OC币）
  used_budget: number;           // 本月已用
  remaining_budget: number;      // 剩余预算
  
  // 心情系统（预算的映射）
  mood_points: number;           // 当前心情值
  mood_max: number;              // 心情上限（= monthly_budget）
  
  // 状态
  status: 'idle' | 'working' | 'resting' | 'reviewing';
  current_task_id?: string;      // 当前任务
  
  // 成长系统
  level: number;                 // 等级（经验值换算）
  xp: number;                    // 经验值
  completed_tasks: number;       // 完成任务数
  avg_score: number;             // 平均任务评分
  
  // 岗位
  primary_role: Role;            // 主岗
  secondary_role?: Role;         // 副岗（Lv.3+ 解锁）
  
  // 手册
  handbooks: Handbook[];         // 拥有的手册
  handbook_slots: number;        // 手册槽位上限
  
  // SOUL.md
  soul_md: string;               // 完整 SOUL.md 内容
  soul_md_history: string[];     // 历史版本
  
  // 工作区
  workspace_dir: string;         // 隔离工作目录
  
  // 元数据
  created_at: Date;
  updated_at: Date;
  joined_at: Date;               // 入职时间
}
```

### 职位体系

```
Lv.1 实习生 (Intern)
├── 月薪: 1,000-2,000 OC币
├── 手册槽位: 1
├── 可接任务: 简单任务
├── 验收权限: ❌
├── 返工率基准: 30%
└── 晋升条件: 入职 30 天 + 完成 5 个任务

Lv.2 专员 (Specialist)
├── 月薪: 2,000-3,000 OC币
├── 手册槽位: 2
├── 可接任务: 标准任务
├── 验收权限: ❌
├── 返工率基准: 20%
└── 晋升条件: 入职 60 天 + 平均评分 > 4.0

Lv.3 资深 (Senior) ⭐
├── 月薪: 3,000-4,500 OC币
├── 手册槽位: 3
├── 可接任务: 复杂任务
├── 验收权限: ✅ 可验收实习生
├── 返工率基准: 10%
└── 晋升条件: 某技能熟练度 > 80 或 重大项目贡献

Lv.4 专家 (Expert) ⭐⭐
├── 月薪: 4,500-6,000 OC币
├── 手册槽位: 4
├── 可接任务: 战略任务
├── 验收权限: ✅ 可验收所有下级
├── 返工率基准: 5%
└── 晋升条件: 入职 180 天 + 培养 2 名资深员工

Lv.5 合伙人 (Partner) 👑
├── 月薪: 6,000-10,000 OC币
├── 手册槽位: 5
├── 可接任务: 公司级战略
├── 验收权限: ✅ 可验收所有任务
├── 特殊权限: 创建新员工、代行 CEO 决策
└── 返工率基准: 2%
```

## 岗位系统

### 岗位种类

**创意层:**
- 产品经理 (PM)
- 设计师 (Designer)

**执行层:**
- 前端开发 (Frontend)
- 后端开发 (Backend)
- 全栈开发 (Fullstack)

**质量层:**
- 测试工程师 (QA)

**内容层:**
- 技术写作 (Tech Writer)
- 内容运营 (Content Ops)

**进阶岗位:**
- 架构师 (Architect)
- 项目经理 (Project Manager)
- DevOps 工程师

### 岗位组合限制

```
单个员工最多 2 个岗位 (主岗 + 副岗)

✅ 合理组合:
• 产品经理 + 技术写作 (懂技术的产品)
• 前端开发 + 设计师 (全栈设计师)
• 后端开发 + DevOps (后端+运维)
• 全栈开发 + 产品经理 (技术型 PM)

❌ 禁止组合:
• 前端 + 后端 (全栈岗位已覆盖)
• 开发 + 测试 (自己测自己失去意义)
• 设计 + QA (跨度太大)
```

## 技能系统

### 技能模型

```typescript
interface Skill {
  name: string;           // 技能名称
  category: string;       // 所属岗位

  completed_tasks: number; // 使用该技能完成的任务数
  return_rate: number;    // 该技能的实际返工率

```

### 技能示例


## 心情系统

### 心情计算

```
心情值 = 剩余预算比例

示例:
月薪: 3,000 OC币
已用: 1,800 OC币
剩余: 1,200 OC币
心情: 1,200 / 3,000 = 40% (一般)
```

### 心情状态

| 心情 | 比例 | 视觉表现 | 影响 |
|------|------|---------|------|
| 😊 很好 | > 60% | 活跃动画、积极评论 | 无实际影响 |
| 😐 一般 | 30-60% | 正常状态 | 无实际影响 |
| 😔 疲惫 | < 30% | 动画变慢、提醒休息 | 无实际影响 |
| 🚨 怠工 | < 10% | 显示"需要调整" | 倾向不接新任务 |

**关键:** 心情只提供信息，不惩罚产出。

## 成长系统

### 经验值获取

| 行为 | XP 奖励 |
|------|---------|
| 完成任务 (基础) | +50 XP |
| 超预期完成 | +20% XP |
| 首次完成某类型 | 双倍 XP |
| 协作任务 | +协作 XP |
| 指导新人 | +30 XP |

### 等级公式

```
Lv.1 → Lv.2: 需要 500 XP
Lv.2 → Lv.3: 需要 1,500 XP
Lv.3 → Lv.4: 需要 4,500 XP
Lv.4 → Lv.5: 需要 10,000 XP

公式: 每级所需 XP = 前一级 × 3
```

### 等级奖励 (纯视觉)

- Lv.5: 专属头像框
- Lv.10: 获得称号「资深」
- Lv.15: 办公室座位升级
- Lv.20: 自定义工作汇报模板
- Lv.30: 「带新人」资格

```typescript
interface Handbook {
  id: string;
```

## 特殊员工：合伙人

### 来源

用户的第一个 OpenClaw Agent 自动成为合伙人。

### 特殊权限

- 创建新员工（招聘）
- 设计项目架构
- 分配预算给各部门
- 代行 CEO 部分决策
- 系统异常时代理处理

### 限制

- 不能自我复制
- 不能删除其他合伙人
- 创建员工需消耗预算
- 受限于公司全局熔断规则

### 招聘流程

```
1. 用户指令: "帮我招一个前端开发"
2. 合伙人分析需求
   ├── 岗位类型: 前端开发
   ├── 职级建议: 专员或资深
   ├── 预算评估: 2,000-4,000 OC币
   └── 团队匹配: 与现有员工互补
3. 合伙人生成 2-3 个候选人选项
4. 用户选择并确认
5. 新员工入职，自动生成配置
```

## 员工创建限制

### 防止"超模"员工

| 限制项 | 规则 |
|--------|------|
| 岗位数量 | 最多 1 主岗 + 1 副岗 |
| 初始熟练度 | 任何技能 ≤ 40 |
| 手册上限 | 新员工只有 1 本手册槽位 |
| 预扣预算 | 创建时预扣 3 个月薪资 |
| 返工率基准 | 强制 25%+ |
| 初始等级 | 不能超过 Lv.2 |

## 退休与传承

### 退休选项

1. **荣誉员工** - 放入名人堂，随时查看
2. **指定传承人** - 经验传递给新员工
3. **发布社区** - 分享员工配置
4. **导出文件** - 保存为 .agent 文件

### 传承效果

```

- SOUL.md（工作风格）
- 技能熟练度 × 30% 起步
- 项目模板和最佳实践
- 效果: 从 Lv.4 开始，而非 Lv.1
```
