### v0.3.0-beta (增强版) - 预计 8-10周
**目标**: 自动Agent管理 + 外网访问 + 架构升级

| 功能 | 工作量 | 说明 | 用户反馈 |

| **精确Token统计** | 3天 | 通过session_status获取真实消耗 | 精确预算必需 |
| **PostgreSQL迁移** | 5天 | 多用户场景，连接池支持 | 架构基础 |

| 功能 | 工作量 | 说明 |

**里程碑**:
- [ ] Partner可自动创建/删除Agent
- [ ] 外网安全访问（API Key + HTTPS）
- [ ] PostgreSQL迁移完成
- [ ] 精确Token统计上线

## 📝 决策记录

### 2026-03-21: v0.3.0 范围重新定义
**决策**: v0.3.0聚焦三大用户核心需求

**新增P0功能**:
1. **Partner自动创建Agent** - 用户明确要求，避免手动配置
2. **外网访问** - 用户需要远程监控公司状态
3. **精确Token统计** - 预算准确性基础

**架构升级**:
- PostgreSQL迁移（支撑多用户和外网场景）

**视觉增强**:
- 像素头像V2（产品差异化）
- 图表可视化（报表实用性）

**推迟到v0.4.0**:
- 子任务系统（多Agent合作基础，需要更完整设计）
- 员工成长系统（游戏化）
- 移动端适配

### 2026-03-21: Agent创建技术方案
**方案**: Partner Agent直接修改openclaw.json

**理由**:
- OpenClaw不提供程序化创建Agent的API
- 文件修改是最直接的方案
- 已有openclaw_config.py读取配置的经验

**缓解风险**:
- 每次修改前自动备份
- 文件锁防止并发写入
- 提供配置回滚功能
- 检测Gateway重启需求

### 2026-03-21: 外网访问安全方案
**方案**: API Key + JWT签名URL

**理由**:
- 简单有效，不需要复杂OAuth
- 适合个人/小团队使用场景
- 易于实现和文档化

**部署建议**:
- 开发/测试：Cloudflare Tunnel（免费，自动HTTPS）
- 生产：自托管 + 自签名证书或Let's Encrypt
