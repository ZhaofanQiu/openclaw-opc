# 预算与熔断机制 (Budget System)

## 核心理念

**虚拟币是 Token 消耗的"包装"，帮助用户直观感知和管理成本。**

- 透明：实时显示 Token 消耗
- 可控：硬边界防止无限消耗
- 有意义：与现实成本挂钩，但不完全绑定

---

## 虚拟币 (OC币)

### 与现实 Token 的关系

```
用户设定换算比例:
1 OC币 = X Token (默认值可配置)

不同模型可设不同系数:
• GPT-4o: 1 OC币 = 100 Token
• GPT-4o mini: 1 OC币 = 1000 Token
• Claude 3.5: 1 OC币 = 120 Token
```

### 资金来源

| 来源 | 说明 |
|------|------|
| **月度拨款** | 每月 1 号重置，等于用户设定的月预算 |
| **任务节省** | 完成得又快又好，返还部分预算 |
| **无其他途径** | 不能刷钱，不能随机获得 |

### 资金用途

| 用途 | 说明 |
|------|------|
| **员工薪资** | 每月固定支出，预扣 3 个月 |
| **任务执行** | 按实际 Token 消耗 |
| **加班费** | 紧急任务临时提升预算 |
| **团建活动** | 恢复员工心情（预算再分配）|

## 预算结构

### 公司级预算

```typescript
interface CompanyBudget {
  monthly_budget: number;      // 月度总预算
  daily_limit: number;         // 日限额 (熔断线)
  
  // 当前状态
  used_today: number;          // 今日已用
  used_this_month: number;     // 本月已用
  remaining: number;           // 剩余预算
  
  // 分配
  employee_budgets: number;    // 员工薪资总额
  project_budgets: number;     // 项目预算总额
  reserve: number;             // 储备金
}
```

### 员工级预算

```typescript
interface EmployeeBudget {
  employee_id: string;
  monthly_salary: number;      // 月薪
  
  // 本月状态
  allocated: number;           // 已分配额度
  used: number;                // 已消耗
  remaining: number;           // 剩余
  
  // 历史
  historical_avg: number;      // 历史平均消耗
  efficiency_score: number;    // 效率评分
}
```

### 任务级预算

```typescript
interface TaskBudget {
  task_id: string;
  estimated_cost: number;      // 预估成本
  actual_cost: number;         // 实际消耗
  
  // 熔断设置
  soft_limit: number;          // 软限制 (80%)
  hard_limit: number;          // 硬限制 (100%)
  fuse_limit: number;          // 熔断线 (150%)
  
  status: 'normal' | 'warning' | 'paused' | 'fused';
}
```

## 熔断机制

### 三级熔断

```
┌─────────────────────────────────────────────┐
│              预算消耗进度                     │
│                                             │
│  0%        80%        100%       150%        │
│   │──────────│──────────│──────────│       │
│   │          │          │          │        │
│  正常        警告        暂停        熔断    │
│  🟢         🟡         🟠         🔴       │
│                                             │
└─────────────────────────────────────────────┘
```

**🟢 正常 (0-80%)**
- 员工正常工作
- UI 显示绿色进度条

**🟡 警告 (80-100%)**
- 系统发送提醒："预算即将耗尽"
- UI 变为黄色
- 员工显示"注意"状态

**🟠 暂停 (100%)**
- 任务自动暂停
- 等待用户决策
- 状态变为"待处理"

**🔴 熔断 (150%)**
- 强制停止任务
- 标记为"预算超支失败"
- 记录异常，用于优化预估

### 熔断后选项


## 时间熔断

### 连续工作保护

```
连续工作时长监控:
├── 1 小时: 正常
├── 2 小时: 提醒 "建议休息"
├── 3 小时: 警告 "即将超时"
└── 4 小时: 强制暂停，保存进度
```

### 协作时间限制

```
多 Agent 协作会议:
├── 最大时长: 30 分钟
├── 最大轮次: 5 轮对话
└── 无结论: 升级人类决策
```

## 预估算法

### 任务成本预估

```python
def estimate_task_cost(task_type, complexity, employee):
    base_cost = {
        'simple': 300,
        'standard': 800,
        'complex': 1500
    }[complexity]
    
    # 员工效率调整
    efficiency = employee.get_efficiency(task_type)
    adjusted = base_cost * (2 - efficiency)  # 效率越高成本越低
    
    # 历史校准
    if employee.historical_avg > 0:
        adjusted = (adjusted + employee.historical_avg) / 2
    
    return {
        'estimated': adjusted,
        'range': [adjusted * 0.7, adjusted * 1.3],
        'confidence': 'medium'  # low/medium/high
    }
```

### 动态校准

```
预估: 800 OC币
实际: 680 OC币
偏差: -15%

系统学习:
├── 记录实际消耗
├── 调整该员工/该类型任务的预估参数
└── 下次预估更准确
```

## 异常处理

### 超支检测

```
检测: 单日消耗 > 日限额 80%
├── 发送预警通知
├── 建议暂停非紧急任务
└── 次日自动恢复

检测: 连续 3 天超支
├── 标记为"预算风险"
├── 合伙人主动提供优化建议
└── 建议调整预算分配
```

### 预算耗尽保护

```
公司预算耗尽:
├── 停止所有新任务创建
├── 保留进行中的任务 (可完成)
├── 仅允许"必要维护"操作
└── 等待次日预算重置或手动追加
```

## 配置选项

```yaml
# config/budget.yaml

company:
  monthly_budget: 10000        # 月度总预算 (OC币)
  daily_limit: 500             # 日限额
  reserve_ratio: 0.2           # 储备金比例 (20%)

employee:
  default_salary: 2000         # 默认月薪
  salary_range: [1000, 10000]  # 薪资范围

task:
  estimation_confidence: 0.3   # 预估置信区间 (±30%)
  soft_limit: 0.8              # 软限制比例
  hard_limit: 1.0              # 硬限制比例
  fuse_limit: 1.5              # 熔断比例

fuses:
  budget_warning: true         # 启用预算警告
  time_limit: true             # 启用时间熔断
  daily_limit: true            # 启用日限额
```
