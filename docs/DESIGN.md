# 产品功能设计 (Product Design)

## 核心理念

### 一人公司操作系统

将 OpenClaw Agent 组织成虚拟公司，让 AI 成为可管理、可成长、可协作的"员工"。

### 设计原则

1. **实用优先** - 游戏化是调味剂，核心是完成项目
2. **安全第一** - 预算熔断、超时保护，防止 Token 无限消耗
3. **可传承** - 员工经验沉淀为资产，可复用、可分享
4. **透明度** - Token 消耗实时可见，成本可控

---

## 三大核心维度

### 1. 员工系统 (Employee)

Agent 作为公司成员，拥有完整的职场属性。

**核心属性:**
- 基本信息：姓名、头像、emoji、职位等级
- 预算：月薪（Token 配额）、已消耗、剩余
- 心情：基于工作量的情绪指标（不影响产出，只影响氛围）
- 技能：岗位熟练度，决定效率
- 手册：工作经验沉淀

**职位体系:**
```
实习生 (Lv.1) → 专员 (Lv.2) → 资深 (Lv.3) → 专家 (Lv.4) → 合伙人 (Lv.5)
```

**特殊角色：合伙人 (Partner)**
- 用户的第一个 OpenClaw Agent 自动成为合伙人
- 协助 CEO 管理公司、招聘员工、处理异常
- 拥有最高权限，但受安全规则约束

### 2. 项目系统 (Project)

**项目生命周期:**
```
立项 → 组队 → 执行 → 里程碑 → 验收 → 复盘
```

**项目属性:**
- 目标、描述、预算上限
- 里程碑（时间线管理）
- 团队成员
- 任务池
- 进度追踪

**项目健康度:**
- 进度：时间线 vs 实际完成
- 士气：团队成员平均心情
- 质量：近期任务评分趋势
- 风险：延期任务、阻塞依赖

### 3. 任务系统 (Task)

**任务类型:**

| 类型 | 图标 | 特点 |
|------|------|------|
| 主线任务 | 🏆 | 项目关键路径，必须完成 |
| 支线任务 | 🎒 | 可选，有额外奖励 |
| 日常任务 | 📅 | 重复性工作 |
| 紧急任务 | 🔥 | 限时，有加成/惩罚 |
| 探索任务 | 🔍 | 研究性质，结果不确定 |

**任务状态流:**
```
待领取 → 进行中 → 待验收 → 已完成
   ↓        ↓         ↓
 可取消   可暂停   可返工
```

**任务分配机制:**
1. 项目负责人拆解任务
2. 系统推荐最佳人选（基于技能匹配）
3. 员工可表达兴趣（申请接任务）
4. CEO/合伙人确认或调整分配

## 预算与成本系统

### 虚拟币 (OC币)

**与现实的关系:**
- 虚拟币是 Token 消耗的"包装"
- 用户设定换算比例（默认 1 OC币 = X Token）
- 不同模型可以有不同的换算系数

**资金来源:**
- 月度预算拨款（等于用户的实际 Token 预算）
- 任务完成节省的返还
- 无其他获取途径（不能刷钱）

**资金用途:**
- 员工薪资（每月固定支出）
- 任务执行（按实际 Token 消耗）
- 加班费/团建（预算再分配）

### 预算熔断机制

**三级熔断:**

| 级别 | 触发条件 | 动作 |
|------|---------|------|
| 警告 | 消耗达到 80% | UI 提醒，员工显示"注意" |
| 暂停 | 消耗达到 100% | 任务暂停，等待决策 |
| 熔断 | 消耗达到 150% | 强制停止，任务失败 |

**熔断后选项:**
- 追加预算（从公司储备调拨）
- 拆分任务（把大任务拆小）
- 换人重做（换一个 Agent 尝试）

### 心情系统 = 预算仪表盘

**核心设计:**
```
心情值 = 剩余预算比例
```

| 心情 | 预算状态 | 视觉反馈 |
|------|---------|---------|
| 😊 很好 | > 60% | 正常动画 |
| 😐 一般 | 30-60% | 偶尔提醒 |
| 😔 疲惫 | < 30% | 显示"需要休息" |

**关键点:** 心情只影响氛围，不影响实际产出质量。

## 工作手册系统

### 三层手册架构

**L1: 岗位通用手册**
- 每个岗位 1 本，不占槽位
- 基础工作规范、流程 checklist

**L2: 专项技能手册**
- 员工最多 2 本
- 决定"能做什么"和"做得多快"

**L3: 项目专属手册**
- 动态生成，随项目归档
- 项目特有规范、踩坑记录

### 手册等级与效果

| 等级 | 熟练度 | 返工率 | 特殊效果 |
|------|--------|--------|---------|
| 入门级 | 0-30 | 25%+ | 基础可用 |
| 初级 | 31-60 | 20% | 自动生成 FAQ |
| 中级 | 61-80 | 15% | 可指导实习生 |
| 高级 | 81-95 | 10% | 可参与评选 |
| 专家级 | 96-100 | 5% | 可发布社区 |

### 手册健康度

**动态返工率追踪:**
- 根据实际任务完成情况更新
- 高等级但高返工率的手册会被警告
- 可替换为社区更优手册

## 员工成长系统

### 三条成长线

**1. 经验值 → 等级 (视觉成就)**
- 完成任务获得 XP
- 等级解锁称号、头像框、座位装饰
- 纯视觉奖励，不影响功能

**2. 技能熟练度 → 效率提升**
- 做某类任务提升对应技能
- 熟练度高 = 返工率低 = 真实省钱

**3. SOUL.md 演化 → 个性形成**
- 基于工作历史自动追加内容
- 形成独特的工作风格

### 晋升机制

**晋升条件 (满足任一):**
- 在岗 N 天 + 平均评分 > 4.0
- 完成重大项目 + CEO 推荐
- 某技能熟练度达到 90

**晋升效果:**
- 解锁新权限（验收、带新人）
- 手册槽位 +1
- 可接更复杂任务

## 互动与氛围系统

### 留言板 (非实时聊天)

**设计:**
- 异步留言，批量处理
- 每人每天限发 3 条
- 支持点赞、简单回复

**触发场景:**
- 工作心得分享
- 求助/问答
- 公司公告

### 事件系统

**触发条件 (仅 IDLE 状态):**
- 心情 < 30%
- 工作时长 > 4 小时
- 任务失败/成功
- 协作冲突

**频率控制:**
- 每个员工每天最多 2 个事件
- 冷却期 4 小时
- 用户可设置"勿扰模式"

### 预算控制

**每日氛围预算池:**
- 固定额度（如 500 OC币）
- 优先保证必要通知
- 有剩余才触发氛围事件

## 资产化与传承

### 员工退休

**触发条件:**
- 用户主动选择
- 员工完成使命（可选机制）

**退休选项:**
1. 保存为"荣誉员工"（名人堂展示）
2. 指定传承人（经验传递）
3. 发布到社区（分享资产）
4. 导出 .agent 文件（本地保存）

### 传承机制

**直接传承:**
- 新员工继承 30% 技能熟练度
- 继承 SOUL.md（工作风格）
- 继承项目模板

**知识萃取:**
- 自动生成《员工手册》（最佳实践）
- 可给所有新员工培训使用
- 可发布社区

### 社区市场 (远期)

**功能:**
- 分享/下载优秀员工配置
- 最佳手册评选
- 跨公司协作（外包任务）

## 安全与限制

### Agent 行为白名单

**允许:**
- 读取工作目录
- 写入工作目录
- 调用 OpenClaw API
- 向特定同事发送消息（限 3 轮）

**禁止:**
- 修改系统配置
- 访问其他员工私有目录
- 创建新 Agent（仅合伙人可）
- 删除历史记录

### 熔断保护

**时间熔断:**
- 连续工作 > 2 小时 → 提醒
- 连续工作 > 4 小时 → 强制暂停

**对话熔断:**
- 协作讨论 > 5 轮 → 升级人类决策
- 单次对话 > 500 Token → 截断

## 差异化亮点

| 特点 | 说明 |
|------|------|
| **真实成本意识** | Token 消耗转化为虚拟币，预算硬边界 |
| **可传承资产** | 员工经验沉淀为手册，可复用分享 |
| **游戏化但不干扰** | 氛围系统不影响实际产出 |
| **合伙人机制** | Agent 辅助管理，降低用户负担 |
| **像素办公室** | 有温度的可视化，区别于冷冰冰的表格 |
