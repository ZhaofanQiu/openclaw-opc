## 🐛 已知Bug

### 高优先级 (P0) - 阻碍使用

| Bug | 描述 | 影响 | 临时解决方案 | 预计修复 |
|-----|------|------|-------------|---------|
| **雇佣员工Internal Error** | Partner绑定Agent后，雇佣员工报500错误 | 无法创建新员工 | 手动检查Partner agent_id绑定 | v0.3.0正式版 |
| **头像设计任务堆积** | 3条头像设计任务pending状态，Partner未处理 | 新员工无头像 | 手动上传头像或忽略 | v0.3.1 |

### 中优先级 (P1) - 影响体验

| Bug | 描述 | 影响 | 临时解决方案 | 预计修复 |
|-----|------|------|-------------|---------|
| **异步消息状态同步延迟** | 消息状态更新后，前端偶尔不刷新 | 需手动刷新页面 | 按F5刷新 | v0.3.1 |
| **Pixel Office偶尔空白** | 快速切换Tab时，像素办公室不渲染 | 视觉体验 | 切换Tab后等待2秒 | v0.3.1 |

### 低优先级 (P2) - 可接受

| Bug | 描述 | 影响 | 临时解决方案 | 预计修复 |
|-----|------|------|-------------|---------|
| **日志重复输出** | structlog某些情况下重复记录 | 日志冗余 | 忽略 | v0.4.0 |
| **API限流提示不友好** | 触发限流时返回英文错误 | 用户体验 | 理解429错误 | v0.3.1 |

## ⚠️ 已知不足

### 功能限制

| 不足 | 描述 | 原因 | 解决方案 |
|------|------|------|---------|
| **单用户系统** | 当前仅支持单用户，无多租户 | SQLite限制 | PostgreSQL已支持，需完善权限 |
| **无持久化WebSocket** | 实时通知依赖轮询 | 简化架构 | 考虑Server-Sent Events |
| **任务无优先级队列** | 高优先级任务不会插队 | MVP简化 | v0.4.0 添加优先级队列 |
| **预算仅月度统计** | 无法查看周/日预算趋势 | 初期设计 | v0.4.0 添加时间维度 |
| **Agent无技能成长** | 员工技能值固定 | 简化模型 | v0.4.0 添加技能经验系统 |

### 性能限制

| 不足 | 描述 | 阈值 | 解决方案 |
|------|------|------|---------|
| **员工数量上限** | Dashboard性能下降 | ~20名员工 | 分页加载 + 虚拟列表 |
| **历史任务查询慢** | 大量任务时查询慢 | ~1000条 | 添加索引 + 归档机制 |
| **头像图片未压缩** | 上传大图片影响加载 | >1MB图片 | 添加图片压缩 |
| **前端包体积大** | 首屏加载慢 | ~2MB | 代码分割 + 懒加载 |

### 安全限制

| 不足 | 描述 | 风险等级 | 解决方案 |
|------|------|---------|---------|
| **API Key明文存储** | 前端localStorage存储 | 中 | 添加HttpOnly Cookie选项 |
| **无操作日志审计** | 无法追踪谁做了什么 | 中 | 添加AuditLog表 |
| **文件上传无类型校验** | 可能上传危险文件 | 低 | 添加MIME类型白名单 |

### 计划集成

#### 1. 技能市场 (Skill Marketplace)

**目标**: 让OpenClaw Skills成为OPC员工的能力

**设计**:
```
User 安装 Skill (e.g., web_search)
    ↓
Skill 注册到 OPC 技能库
    ↓
创建员工时选择 "具备 web_search 技能"
    ↓
任务分配时匹配具备对应技能的员工
```

**实现步骤**:
1. 扫描 `.openclaw/skills/` 目录
2. 解析 SKILL.md 获取技能信息
3. 同步到 OPC 技能数据库
4. 员工-技能关联表

**时间线**: v0.4.0

---

#### 2. 记忆共享 (Memory Sharing)

**目标**: Agent之间可以共享上下文

**当前问题**:
- 每个Agent独立memory，无法协作
- Partner和员工之间信息不互通

**设计方案**:
```
公司级 Memory (共享)
├── 项目文档
├── 代码规范
├── API文档
└── 会议纪要

员工级 Memory (私有)
├── 个人任务历史
└── 个人偏好
```

**技术实现**:
```python
# 创建公司级memory文件
# /workspace/memory/company/project_docs.md

# Agent启动时自动加载
system_prompt += load_company_memory()
```

**时间线**: v0.4.0

---

#### 3. 跨Agent对话 (Agent-to-Agent Chat)

**目标**: Agent可以直接对话协作

**场景**:
- 前端Agent询问后端Agent API接口
- 设计师询问产品经理需求细节

**实现**:
```
员工A发送消息给员工B
    ↓
OPC 转发消息到员工B的OpenClaw Agent
    ↓
员工B回复
    ↓
OPC 转发回员工A
```

**API设计**:
```
POST /api/agents/{from_id}/message/{to_id}
{
  "content": "询问内容",
  "context": {相关任务上下文}
}
```

**时间线**: v0.4.0

## 🏗️ 技术债务

### 需要重构的代码

| 模块 | 问题 | 重构方案 | 优先级 |
|------|------|---------|--------|
| **前端路由** | 所有代码在单个HTML | 使用Vue Router | P1 |
| **API客户端** | 重复的fetch逻辑 | 封装统一API层 | P2 |
| **状态管理** | 全局变量混乱 | Pinia/Vuex | P2 |
| **后端Service** | 部分代码重复 | 提取公共基类 | P2 |
| **数据库查询** | 缺少优化 | 添加索引 | P1 |
